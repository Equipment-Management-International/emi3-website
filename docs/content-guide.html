EMI Website — Content Management Guide

How to manage content on equipmentmanagement.com.au using Contentful and Claude Desktop.

How the Website Works

The EMI website is a static site — it's built once and served as plain HTML files. Content changes require a rebuild, which happens automatically.

What Lives Where

Claude Desktop can help you change both — just ask. The split is about what kind of content it is, not who can change it.

📝

Contentful (CMS)

Repeatable, structured content

Content that follows a template and gets created regularly. Each item has the same fields (title, date, body, tags, image, etc.).

  • ✓ News articles & blog posts
  • ✓ Article featured images
  • ✓ Testimonials & case studies
  • ✓ Product features & descriptions

Think: "I'll be adding more of these over time."

⚙️

Astro (Code)

Layout, design, one-off pages

The structure and look of the site. Pages that are unique and don't follow a repeating pattern.

  • ✓ Page layouts & design
  • ✓ Navigation & footer
  • ✓ Homepage sections
  • ✓ Contact form
  • ✓ Hero images & logos

Think: "This is set up once and changes occasionally."

Not sure where something belongs?

Just ask Claude — it knows the site structure and will put things in the right place. You don't need to decide.

Content Workflows with Claude Desktop

Writing a News Article

1

Ask Claude to write the article

Example: "Write a news article about our new partnership with BHP for the Jimblebar site. Tag it as mining and news."

2

Review the draft

Claude creates it as a draft in Contentful. Ask to see it, request changes, or edit directly in the Contentful web app.

3

Publish when ready

Say "publish that article" or publish it in Contentful. This triggers an automatic rebuild — the article goes live within a few minutes.

Adding Images

Images from the web

Give Claude the image URL and it uploads directly to Contentful.

Example: "Use this image for the article: https://example.com/photo.jpg"

Local images (photos, screenshots)

Upload them directly in the Contentful web app (Media section, drag & drop), then tell Claude to use it.

Example: "Attach the image I just uploaded called site-photo.jpg to the article"

Draft vs Published

flowchart LR D["📄 Draft\n(not visible on site)"] -->|"publish"| P["✅ Published\n(live on site)"] P -->|"unpublish"| D style D fill:#fff3cd,stroke:#C5960C,stroke-width:2px,color:#333 style P fill:#d4edda,stroke:#28a745,stroke-width:2px,color:#333
  • Draft: Content exists in Contentful but is NOT on the live website. Safe to edit and review.
  • Published: Content is live. Publishing triggers Netlify to rebuild the site (takes 1-3 minutes).
  • Unpublish: Removes content from the live site without deleting it. You can republish later.

Getting Started

One-time Setup

  1. Install Claude Desktop
  2. Get the .env file from your developer (contains API keys)
  3. Place it in the project root folder (same folder as .env.example)

Each Time You Want to Edit Content

🍎 macOS

Open Terminal in the project folder, then run:

./launch-claude.sh

🪟 Windows

Open PowerShell in the project folder, then run:

.\launch-claude.ps1

This loads your API keys and opens Claude Desktop with Contentful access. You cannot just open Claude Desktop from the dock/Start Menu — it won't have the keys.
